📌 نبذة عن المشروع

ما هو LoraTrak؟

نظام ذكي مصمم لحل مشكلة تتبع الحافلات في المناطق التي تعاني من ضعف تغطية الإنترنت

🎯

المشكلة

في العديد من المدن والمناطق الريفية، لا تتوفر تغطية إنترنت مستقرة، مما يجعل أنظمة التتبع التقليدية التي تعتمد على GPS + 4G/WiFi غير فعالة. الركاب لا يعرفون متى ستصل الحافلة، مما يسبب هدراً كبيراً للوقت.

💡

الحل

LoraTrak يستخدم تقنية LoRa للاتصال بعيد المدى بدون إنترنت، مع شبكة Mesh لتمرير البيانات بين العقد. يتم حساب المسافة والوقت المتبقي محلياً وعرضه على شاشات LCD وواجهة ويب عند توفر الاتصال.

👥

الفئة المستهدفة

  • شركات النقل العمومي
  • إدارات النقل المدرسي والجامعي
  • النقل بين المدن في المناطق الريفية
  • المشاريع الذكية والمدن الذكية
🏆

القيمة المضافة

  • تقليل وقت الانتظار بنسبة تصل إلى 60%
  • خفض تكاليف التشغيل (لا حاجة لبطاقات SIM)
  • تحسين رضا الركاب بشكل ملحوظ
  • بيانات تحليلية لتحسين الخطوط

كيف يعمل النظام؟

من قراءة GPS إلى عرض البيانات على الشاشة، كل خطوة تتم تلقائياً

1

قراءة بيانات GPS

تقوم وحدة GPS داخل عقدة الحافلة بقراءة الإحداثيات الجغرافية بشكل دوري كل 5 ثوانٍ. يتم استخراج البيانات من جمل NMEA وتنقيتها من الضوضاء.

2

حساب السرعة اللحظية

باستخدام الفرق بين قراءتين GPS متتاليتين والزمن بينهما، يتم حساب السرعة اللحظية. يتم تطبيق فلتر Moving Average لتنعيم القراءات.

3

تغليف البيانات في Packet

يتم تجميع البيانات في حزمة تحتوي على: معرف الحافلة، الإحداثيات، السرعة، الطابع الزمني، ورقم تسلسلي للتحقق من سلامة البيانات.

4

الإرسال عبر LoRa Mesh

يتم إرسال الحزمة عبر شبكة Mesh. إذا لم تكن المحطة في المدى المباشر، يتم تمرير البيانات عبر العقد الوسيطة تلقائياً حتى تصل.

5

حساب المسافة والوقت (Haversine)

محطة الاستقبال تطبق معادلة Haversine لحساب المسافة بين الحافلة والمحطة. ثم تقسم المسافة على السرعة للحصول على وقت الوصول المتوقع.

6

عرض النتائج للمسافرين

يتم عرض البيانات على شاشة LCD في المحطة. في نفس الوقت، إذا توفر Wi-Fi، يتم إرسال البيانات إلى خادم الويب لعرضها على الخريطة التفاعلية.

🔗 بنية الشبكة

شبكة Mesh Network

شبكة لا مركزية حيث كل عقدة يمكنها إرسال واستقبال وتمرير البيانات

عقدة LoRa محطة

🔀 ما هي شبكة Mesh؟

شبكة Mesh هي بنية اتصال لا مركزية حيث كل جهاز يتصل بعدة أجهزة أخرى، مما يخلق مسارات متعددة لنقل البيانات. إذا فشل مسار واحد، يتم إيجاد مسار بديل تلقائياً.

🔁 تمرير البيانات (Relay)

عندما لا تكون الحافلة في المدى المباشر للمحطة، تقوم العقد الوسيطة بتمرير البيانات تلقائياً. كل عقدة تضيف معرفها كـ "hop" لمنع الحلقات.

🛡️ الموثوقية والتسامح مع الأعطال

بفضل تعدد المسارات، الشبكة تتحمل عطل أي عقدة وسيطة. نسبة تسليم البيانات تتجاوز 99.2%.

↔️ قابلية التوسع

يمكن إضافة عقد جديدة بسهولة بدون إعادة تكوين الشبكة. كل عقدة جديدة تزيد من تغطية الشبكة وتدعم عشرات الحافلات في نفس الوقت.

📐 المحرك الحسابي

معادلة Haversine

المعادلة الرياضية التي تحسب المسافة بين نقطتين على سطح الكرة الأرضية

∑ المعادلة الرياضية

a = sin²(Δφ/2) + cos(φ₁)·cos(φ₂)·sin²(Δλ/2)
──────────────────────────────
c = 2·atan2(√a, √(1−a))
──────────────────────────────
d = R · c
  • φ = خط العرض (Latitude)
  • λ = خط الطول (Longitude)
  • R = نصف قطر الأرض (6371 كم)
  • d = المسافة بين النقطتين

⚠️ لماذا Haversine؟

معادلة Haversine أكثر دقة من حساب المسافة الإقليدية البسيطة لأنها تأخذ في الاعتبار انحناء الأرض. الخطأ باستخدام الطريقة الإقليدية قد يصل إلى 0.5%، بينما Haversine توفر دقة تتجاوز 99.9%.

haversine.cpp 
// Haversine Formula
#define R 6371000

double haversine(
    double lat1, double lon1,
    double lat2, double lon2
) {
    double dLat = toRad(lat2 - lat1);
    double dLon = toRad(lon2 - lon1);

    double a = sin(dLat/2)*sin(dLat/2) +
        cos(toRad(lat1))*cos(toRad(lat2))*
        sin(dLon/2)*sin(dLon/2);

    double c = 2*atan2(
        sqrt(a), sqrt(1-a));

    return R * c; // meters
}

// ETA in minutes
int calcETA(double dist_m, double spd) {
    if (spd < 1) return -1;
    return (int)((dist_m/1000.0)/spd*60);
}

الأسئلة الشائعة

هل يحتاج النظام إلى اتصال إنترنت؟

+
لا، النظام يعمل بشكل كامل بدون إنترنت. الاتصال بين الحافلات والمحطات يتم عبر LoRa. واجهة الويب اختيارية وتعمل فقط عند توفر Wi-Fi أو Ethernet في المحطة.

ما هو أقصى مدى للاتصال؟

+
مدى الاتصال المباشر يصل إلى 15 كم في الأماكن المفتوحة. في المناطق الحضرية ينخفض إلى 2-5 كم. مع شبكة Mesh، يمكن تغطية مسافات أطول بكثير عبر العقد الوسيطة.

كم عدد الحافلات التي يمكن تتبعها؟

+
نظراً لأن كل حافلة ترسل بياناتها كل 5 ثوانٍ وحجمها صغير جداً (~30 بايت)، يمكن دعم عشرات الحافلات. مع Channel Management يمكن الوصول إلى أكثر من 100 حافلة في نفس المنطقة.

ما هي تكلفة تصنيع عقدة واحدة؟

+
تكلفة مكونات عقدة الحافلة تتراوح بين 15-25 دولار: ESP32 (~$4)، LoRa SX1276 (~$5)، GPS Neo-6M (~$4)، بطارية Li-Po (~$3)، العلبة والموصلات. محطة الاستقبال مع الشاشة تكلف ~35-45 دولار.

كيف يتم تزويد الأجهزة بالطاقة؟

+
عقدة الحافلة تعمل ببطارية ليثيوم (2000mAh) تدوم 72 ساعة، أو من كهرباء الحافلة (12V/24V). محطة الاستقبال تعمل بالطاقة الشمسية أو من الشبكة الكهربائية.

هل البيانات مشفرة وآمنة؟

+
نعم، يتم تشفير جميع البيانات باستخدام AES-128 قبل الإرسال. كل حزمة تحتوي على رقم تسلسلي وCRC Checksum. Network ID يمنع التدخل من أجهزة خارجية.

هل يمكن استخدامه لتتبع أشياء أخرى؟

+
بالتأكيد! النظام مصمم ليكون مرناً. يمكن استخدامه لتتبع الشاحنات، السيارات، أو الأصول الثمينة. يمكن إضافة Geofencing وإنذارات عند مغادرة منطقة جغرافية محددة.
